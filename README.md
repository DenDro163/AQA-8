# **План тестирования записи на обучение по профессии "Тестировщик ПО".**
## 1. Введение.
* Будет проводиться внедрение автоматизации тестирования возможности записи на курс "Тестировщик ПО" на сайте  ООО «Нетология». Запись проводиться через форму на странице профессии. Сайт предназначен для широкого круга пользователей, желающих получить дополнительное образование. Пользователи могуть быть как зарегистрированными, так и незарегистрированными.
* Критериями успешного тестирования являются:
  * Возможность записаться на курс при корректном заполнении формы записи
  * Невозможность записи и получение уведомлений о некорректном заполнении полей формы
## 2. Виды тестирования.
* Функциональное тестирование формы записи
* Негативное тестирование полей формы записи
* Тестирование граничных значений поля телефон

### Тестовые сценарии.
#### Сценарий HappyPath RegisrationCard (неавторизованный пользователь):
1. Предусловие 1.
1. Заполнить поле Имя - имя должно состоять из букв кириллицы или латиницы.
1. Заполнить поле Телефон - поле должно быть заполнено по шаблону +7 (999) 999-99-99.
1. Заполнить поле e-mail (поле не должно содержать кавычки, двоеточие, фигурные и квадратные скобки).
1. Нажать "Записаться"
1. Ожидаемый результат - сообщение об успешной записи
#### Сценарий EmptyField RegisrationCard (неавторизованный пользователь):
1. Предусловие 1.
1. Нажать "Записаться".
1. Ожидаемый результат - во всех полях сообщение "Обязательное поле", запись не производиться.
#### Сценарий WrongСhar RegisrationCard NameField (неавторизованный пользователь):
1. Предусловие 1.
1. Заполнить поле Имя - имя должно содержать цифры, знаки (не использовать дефис).
1. Нажать "Записаться".
1. Ожидаемый результат - сообщение "Должно состоять из букв", запись не производиться.
#### Сценарий WrongСhar RegisrationCard TelephoneField (неавторизованный пользователь):
Предусловие 1
1. Заполнить поле Телефон - поле должно содержать буквы, символы (не использовать скобки, знак плюс).
1. Нажать "Записаться".
1. Ожидаемый результат - сообщение "Номер в формате +9 (999) 999-99-99", запись не производиться.
#### Сценарий ShortNumber RegisrationCard TelephoneField (неавторизованный пользователь):
1. Предусловие 1.
1. Заполнить поле Телефон - поле должно содержать менее 11 цифр.
1. Нажать "Записаться".
1. Ожидаемый результат - сообщение "Номер в формате +9 (999) 999-99-99", запись не производиться.
#### Сценарий LongNumber RegisrationCard TelephoneField (неавторизованный пользователь):
1. Предусловие 1.
1. Заполнить поле Телефон - поле должно содержать более 11 цифр.
1. Нажать "Записаться".
1. Ожидаемый результат - сообщение "Номер в формате +9 (999) 999-99-99", запись не производиться.
#### Сценарий WrongСhar RegisrationCard EmailField (неавторизованный пользователь):
1. Предусловие 1.
1. Заполнить поле Почта - поле должно содержать кавычки, двоеточие, фигурные и квадратные скобки, более или ни одного знака «сommercial at», более двух точек после знака «сommercial at».
1. Нажать "Записаться".
1. Ожидаемый результат - сообщение "Неверный email", запись не производиться.
#### Сценарий RegisrationCard AuthorizedUser (авторизованный пользователь):
1. Предусловие 2.
1. Ожидаемый результат - поля "Имя" и "Телефон" заполнены, поле "email" отсутсвует.
1. Нажать записаться.
1. Ожидаемый результат - сообщение об успешной записи.
## 4. Перечень используемых инструментов.
* Java 11 - версия в которой разрабатывается ПО.
* IntelliJ IDEA - удобная среда разработки для JAVA.
* Gradle - популярная система автоматической сборки.
* JUnit 5 - вложенные и параметризованные тесты.
* Selenide - управление браузером и создание автоматических скриншотов.
* Faker - генерация пользовательских данных.
* Docker - система для быстрого развёртывания и перезапуска базы данных.
* Git - система контроля версий.
* GitHub - удалённое хранилище проекта.
* Reporting.Allure - автоматическое создание отчётов.
* GitHub.Actions - система CI CD.
## 5. Перечень необходимых разрешений.
* Разрешение ответственного лица организации ООО «Нетология» на проведение тестирования.
* Доступ к API (токены / документация).
* Доступ к БД.
* ac/dc power up.
## 6. Перечень и описание возможных рисков при автоматизации.
* Высокая стоимость используемых инструментов.
* Большие сроки написания тестов, возможна потеря актуальности некоторых тестов (банер).
* Необходимость поддержки тестов.
## 7. Перечень необходимых специалистов для автоматизации.
